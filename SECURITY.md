# Security Policy

## Supported Versions
The following versions of this project are currently being supported with security updates:

| Version | Supported |
|----------|------------|
| main     | ✅ Active  |

Older or archived branches are **not maintained** for security fixes.

---

## Reporting a Vulnerability
If you discover a security vulnerability, please follow the responsible disclosure process below:

1. **Do not create a public GitHub issue.**
2. **Privately report** the vulnerability by emailing: `aaditya.r@nyu.edu`
3. Include:
   - Steps to reproduce
   - A clear description of the impact
   - Any relevant logs or proof of concept

Reports will be acknowledged within **48 hours**, and you’ll receive a response on:
- The assessment of the issue
- Expected timeline for a fix
- Credit acknowledgment (if desired)

---

## Handling Security Updates
Once a vulnerability is confirmed:
- A patch will be created on a private branch.
- After testing and verification, it will be merged into `main`.
- A release note will summarize the fix and credit the reporter if applicable.
